# Фаза 2 — фото-карусели, видеогалерея, отдельное аудио

Новая возможность: бот теперь умеет скачивать **фото-посты / карусели /
слайдшоу** (раньше не умел вообще — был только режим video-only с
`--no-playlist`). Плюс 2 новые настройки в меню ⚙️ (личный чат).

## Новые настройки (личный чат; в группах — поведение по умолчанию)

### 🎞 Видеогалерея (`gallery_mode`) — переключатель
- `Альбом` (по умолчанию) — фото карусели отправляются альбомом (media group).
- `Видео-слайдшоу` — из фото собирается видео-слайдшоу под музыку поста (ffmpeg).

### 🔊 Аудио (`audio_mode`) — переключатель
- `Не отправлять` (по умолчанию).
- `Отдельным файлом` — фоновое аудио поста дополнительно присылается как .mp3.

### Комбинации (для TikTok photo+audio и др.)
| 🎞 Видеогалерея | 🔊 Аудио | Результат |
|---|---|---|
| Альбом | Не отправлять | альбом фото |
| Альбом | Отдельным файлом | альбом фото + аудио .mp3 (всё раздельно) |
| Видео-слайдшоу | Не отправлять | слайдшоу-видео с музыкой |
| Видео-слайдшоу | Отдельным файлом | слайдшоу-видео + аудио .mp3 |

## Как это работает
1. **Детект фото-поста.**
   - Instagram: перед загрузкой делается `get_info`; если пост — карусель без
     видео (`is_photo_post`), скачиваем всю галерею.
   - TikTok: обычная видео-загрузка слайдшоу падает (нет видео-формата) →
     автоматический fallback на загрузку галереи. Так не тратим время на лишний
     `get_info` для обычных видео.
2. **Загрузка галереи** (`download_gallery`): yt-dlp без `--no-playlist`
   скачивает все элементы во временную папку; аудио берётся отдельным файлом или
   до-извлекается `bestaudio → mp3`.
3. **Слайдшоу** (`build_slideshow`, ffmpeg): каждое фото масштабируется в портрет
   1080×1920 (с полями, без обрезки), длительность кадра подгоняется под длину
   песни (`ffprobe`), `-shortest` синхронизирует. Если аудио нет — 3 c/кадр.
4. **Отправка**: альбом (media group, чанки по 10) или слайдшоу-видео;
   опционально аудио отдельным файлом. Подпись (Фаза 1) и описание работают как
   обычно.

## Защита и фолбэки
- Слишком большие файлы (> лимита Telegram) пропускаются; если альбом не
  отправился — фолбэк на отправку как документы; если слайдшоу не собралось —
  фолбэк на альбом фото.
- Временная папка галереи всегда чистится (`cleanup_gallery`).

## Зависимости на сервере
- `ffmpeg` и `ffprobe` должны быть в PATH (уже используются в `split_video`).
  Можно переопределить через env `FFMPEG_BIN` / `FFPROBE_BIN`.

## ⚠️ Важно — нужна проверка на живых ссылках
Извлечение фото/аудио зависит от того, как их отдаёт yt-dlp для конкретной
платформы, и **меняется со временем**. Я проверил локально сборку слайдшоу,
миграцию БД, логику отправки и все режимы на мок-данных, но **реальные TikTok/
Instagram фото-посты протестируй на боте** и скажи, если что-то не подхватывается
— подправлю extractor-логику.

## Изменённые файлы
- `database/models.py` — миграция: `gallery_mode TEXT DEFAULT 'photos'`,
  `audio_mode TEXT DEFAULT 'off'` в `users` (идемпотентно).
- `database/db.py` — `get_user_settings` + `set_user_gallery_mode`/
  `set_user_audio_mode`; `VALID_GALLERY_MODES`/`VALID_AUDIO_MODES`.
- `services/downloader.py` — `GalleryResult`, `download_gallery`,
  `build_slideshow`, `is_photo_post`, `cleanup_gallery`, `_extract_audio`,
  `_audio_duration`, константы `FFMPEG`/`FFPROBE`/`*_EXTS`.
- `handlers/download.py` — детект фото-поста + fallback; `process_gallery`;
  `_resolve_caption_config` (вынесена общая логика подписи).
- `handlers/settings.py` — колбэк `uset:cycle:gallery` / `uset:cycle:audio`.
- `keyboards/inline.py` — 2 новых пункта меню (переключатели).
- `services/i18n.py` — ключи en/ru/uk (set_opt_gallery/audio, gal_*, aud_*).

## Миграция
Автоматическая при старте (`init_db`). Новые колонки добавляются, если их нет.
