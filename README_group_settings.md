# smdownbot — Настройки по группам (per-group settings)

Эта фича: когда пользователь добавляет бота в группу/канал — этот чат
появляется у него в боте (в личке), и по тапу можно настроить под конкретную
группу:

1. **Хештеги источника** в подписи (вкл/выкл)
2. **Канал/автор источника** в подписи (вкл/выкл)
3. **Режим загрузки** (спрашивать / видео / аудио)
4. **Удалять ссылку юзера** — удалить сообщение со ссылкой после того, как бот
   прислал загрузку

## Как пользоваться
- Добавьте бота в группу/канал (как обычно). Бот сразу запоминает чат и кто его
  добавил.
- В личке с ботом откройте **⚙️ Настройки → 👥 Мои группы** (или команда
  **/mygroups**).
- Выберите группу из списка → откроется панель с 4 настройками выше. Тапайте,
  чтобы менять.
- Настраивать может тот, кто добавил бота, **или** текущий админ этой группы.

## Важные нюансы
- **Подписи в группах** теперь берутся из настроек группы (а не из личных
  настроек юзера). Глобальные админ-флаги (`feature_show_tags`,
  `feature_show_source_channel`) по-прежнему работают как «принудительно включить
  для всех» (логическое ИЛИ с настройкой группы).
- **Режим загрузки в группе** теперь индивидуальный для каждой группы. При первой
  регистрации группа наследует текущий глобальный `group_download_mode`, дальше
  работает её собственная настройка.
- **Удаление ссылки юзера** требует, чтобы у бота были права админа на удаление
  сообщений в группе. Если прав нет — бот просто молча не удаляет (ошибки нет).
- **Группы, добавленные ДО этого обновления** (когда `my_chat_member` ещё не
  отслеживался): теперь определяются автоматически. Бот лениво регистрирует любой
  чат/канал, как только видит в нём активность (любое сообщение/пост), а в списке
  «Мои группы» показываются не только те, куда вы сами добавили бота, но и любые
  чаты, где **вы сейчас админ/создатель**. При первом открытии настроек такая
  «бесхозная» группа закрепляется за вами (added_by), чтобы дальше открываться
  мгновенно. Telegram не даёт API «список всех чатов бота», поэтому чат должен
  хотя бы раз проявить активность, чтобы попасть в список.

## Что изменилось в коде
- `database/models.py` — новая таблица `group_chats` (идемпотентная миграция,
  старые данные не трогает).
- `database/db.py` — `ensure_group_chat`, `set_group_inactive`,
  `get_group_settings`, `list_user_groups`, `toggle_group_flag`
  (с белым списком колонок — защита от SQL-инъекций), `set_group_download_mode`.
- `handlers/groups.py` (новый) — отслеживание добавления/удаления бота через
  `my_chat_member`, команда `/mygroups`, роутер колбэков `gset:` с проверкой прав,
  middleware `GroupRegistrationMiddleware` (ленивая регистрация чатов по активности)
  и определение групп по статусу админа.
- `database/db.py` — добавлены `list_active_groups` и `claim_group`.
- `bot.py` — подключён `groups.router` и middleware регистрации групп, в
  `allowed_updates` добавлены `my_chat_member`, `channel_post`, `edited_message`,
  команда `/mygroups` в меню.
- `handlers/download.py` — в группах используется режим загрузки группы;
  подписи и удаление ссылки берутся из настроек группы.
- `keyboards/inline.py` — клавиатуры списка групп / панели группы / выбора режима;
  кнопка «👥 Мои группы» в меню настроек.
- `services/i18n.py` — новые строки (en/ru/uk).

## Установка
Распакуйте архив поверх проекта (с сохранением структуры папок), заменив файлы.
Применяйте архивы по порядку — этот последним. Перезапустите бота — миграция
таблицы выполнится автоматически при старте.

> Примечание: `smdownbot_group_settings.diff` — это полный набор изменений
> относительно оригинала (включает все предыдущие доработки); эта поставка
> добавляет именно настройки по группам.
