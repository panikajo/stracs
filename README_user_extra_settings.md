# Новые настройки пользователя — Фаза 1

Добавлены 3 новые персональные настройки в меню ⚙️ (личный чат). Работают
только в приватном чате с ботом; в группах поведение прежнее (рулят настройки
группы).

## Что добавлено

### 1. 💧 Водяной знак TikTok (вкл/выкл)
- Управляет тем, скачивать ли TikTok-видео **с родным водяным знаком** TikTok
  (с @автором) или **без него**.
- По умолчанию **выкл** → как раньше, без водяного знака (чистая копия).
- Реализация: при вкл yt-dlp пробует формат `download` (копия с водяным
  знаком), при недоступности — fallback на `best`.
- ⚠️ Набор форматов TikTok меняется со временем. Если копия с водяным знаком
  перестанет подтягиваться — правится одна строка в `services/downloader.py`
  (см. комментарий рядом с `opts["format"] = "download/best"`).

### 2. 🏷 Подпись под контентом (выбор)
Строка-кредит под контентом. Варианты:
- `Источник + Via (автор)` — как сейчас (🎬 Джерело ✦ Via) — **по умолчанию**
- `Источник` — только ссылка на источник
- `+ Источник` — заглушка под будущий `/custom_caption` (пока = «Источник»)
- `Автор` — только 👤 @автор (ссылка на профиль)
- `Отключено` — без строки-подписи

### 3. 📝 Описание (выбор)
Как отправлять текст/описание поста:
- `Выключено` — не показывать — **по умолчанию**
- `Отдельно` — отдельным сообщением под контентом (цитатой)
- `С контентом` — добавить в подпись к медиа
- `Цитатой` — добавить в подпись как `<blockquote>`
- `Страна` — показать страну/гео загрузки (когда платформа отдаёт это поле)
- ⚠️ Гео (`Страна`) и описание у TikTok отдаются **не всегда** — если поля нет,
  бот просто ничего не добавит (без ошибки).

## Изменённые файлы
- `database/models.py` — миграция: новые колонки `tiktok_watermark`,
  `description_mode`, `caption_mode` в таблице `users` (идемпотентно).
- `database/db.py` — `get_user_settings` отдаёт новые поля; новые сеттеры
  `set_user_description_mode`, `set_user_caption_mode`; `tiktok_watermark`
  добавлен в `USER_FLAG_COLUMNS`; списки допустимых значений.
- `services/downloader.py` — `DownloadResult.description` / `.geo`; параметр
  `download(..., watermark=...)`; печать `%(description)j` и `%(location)j`;
  выбор формата TikTok с/без водяного знака.
- `handlers/download.py` — `build_caption()` переписан под `caption_mode` /
  `desc_mode`; персональные настройки применяются в личном чате; отправка
  описания отдельным сообщением; передача `watermark` в загрузку.
- `keyboards/inline.py` — 3 новых пункта в меню настроек + два пикера
  (`settings_caption_keyboard`, `settings_description_keyboard`).
- `handlers/settings.py` — обработка колбэков `uset:caption/desc/setcaption/setdesc`.
- `services/i18n.py` — новые ключи для en/ru/uk.

## Миграция
Автоматическая при старте (`init_db`). Новые колонки добавляются, если их нет;
существующие данные не трогаются.

## Не входит (Фаза 2)
- Видеогалерея (фото-карусель → видео-слайдшоу под музыку).
- Аудио / раздельная отправка фото+аудио для TikTok photo-mode.
- Сейчас бот вообще не качает фото-карусели — это отдельная новая возможность.
