# /stats в группе → статистика загрузок по платформам

## Что добавлено
Команда `/stats` в группе/чате теперь показывает количество загрузок
**именно в этом чате**, сгруппированное по платформам, например:

```
📊 Загрузки в этом чате

📅 Сегодня
🔴 YouTube — 3
📸 Instagram — 4
🎵 TikTok — 10
➖ Итого: 17

🗂 За всё время
🔴 YouTube — 3000
📸 Instagram — 4000
🎵 TikTok — 10
➖ Итого: 7010
```

- В **личке** `/stats` по-прежнему показывает персональные лимиты (как было).
- Считаются только реально завершённые загрузки.
- Учёт ведётся по `chat_id`, так что у каждой группы своя статистика.

## Технические детали
1. **`database/models.py`** — миграция: в таблицу `downloads` добавлена колонка
   `chat_id` (+ индекс `idx_downloads_chat`). Миграция идемпотентна, старые
   строки сохраняются (у них `chat_id = NULL`).
2. **`database/db.py`**
   - `record_download(...)` получил параметр `chat_id`.
   - Новая функция `get_chat_download_stats(chat_id)` — возвращает счётчики
     по платформам за сегодня и за всё время.
3. **`handlers/download.py`** — во все 3 места записи загрузки теперь
   передаётся `chat_id` (группа или личка).
4. **`handlers/start.py`** — `/stats` в группе рендерит статистику чата;
   платформы берут иконку/название из `services/platform.py`.
5. **`services/i18n.py`** — новые строки `chat_stats_*` (en/ru/uk).

## Важно про старые загрузки
Колонка `chat_id` появляется только сейчас, поэтому загрузки, сделанные
**до** обновления, не привязаны к чату (`chat_id = NULL`) и в статистику
конкретной группы не попадут. Счётчики начнут расти с момента установки.

## Установка
Распакуйте поверх проекта (это продолжение предыдущих архивов — все
изменения из них уже включены в эти файлы). Новых зависимостей нет.
При первом запуске миграция добавит колонку автоматически.
